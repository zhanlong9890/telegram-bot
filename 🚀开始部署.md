## 🎯 快速部署（3步完成）

### 第 1 步：推送到 GitHub

如果代码还没有推送到 GitHub：

```bash
# 初始化 Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "准备部署到 Railway"

# 在 GitHub 创建新仓库，然后推送
git remote add origin https://github.com/你的用户名/你的仓库名.git
git branch -M main
git push -u origin main
```

### 第 2 步：部署到 Railway

1. **访问 Railway**
   - 打开浏览器，访问：https://railway.app
   - 点击右上角 **"Login"**

2. **登录**
   - 选择 **"Login with GitHub"**
   - 授权 Railway 访问你的 GitHub

3. **创建项目**
   - 点击 **"New Project"**
   - 选择 **"Deploy from GitHub repo"**
   - 在列表中找到并选择你的代码仓库
   - 点击 **"Deploy Now"**

4. **配置环境变量**
   - 在项目页面，点击 **"Variables"** 标签
   - 点击 **"+ New Variable"**
   - 添加：
     - **名称**：`BOT_TOKEN`
     - **值**：`8468223563:AAGwZNfTT97sGhXaqba9ZBb-w1OznieV67k`（你的机器人Token）
   - 点击 **"Add"** 保存

5. **等待部署**
   - Railway 会自动安装依赖并启动机器人
   - 查看日志：点击部署卡片 → **"View Logs"**
   - 看到 `机器人启动中...` 说明成功！

### 第 3 步：测试机器人

1. 在 Telegram 中找到你的机器人
2. 发送 `/start` 命令
3. 如果收到回复，部署成功！🎉

## 🐛 遇到问题？

### 机器人不响应
1. 检查 `BOT_TOKEN` 是否正确
2. 查看 Railway 日志是否有错误
3. 确认机器人已启动（日志中看到 "机器人启动中..."）

### 部署失败
1. 查看部署日志中的错误信息
2. 确认 `bot.py` 文件存在且正确
3. 确认 `requirements.txt` 格式正确
